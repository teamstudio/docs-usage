    カタログ

    Catalog モジュールは、Domino サーバーのカタログおよびアドレス帳から情報を取り込みます。このモジュールは、サーバー上のすべてのデータベース、テンプレート、ユーザーのマスターリストとして機能します。

    Catalog は、サーバースキャンを実行するたびに更新されます。Teamstudio Usage の設定が完了していれば、File メニューから Scan Servers を選択することで、いつでも Catalog を更新できます。この操作により、Catalog データと Usage データの両方が更新されます。

    Note

    Teamstudio Usage は、各スキャン対象サーバーの catalog.nsf からカタログデータを読み取ります。サーバーのデータディレクトリを直接スキャンするわけではありません。そのため、データベースの追加や削除などサーバー側で行われた変更は、サーバーで catalog タスクが実行され catalog.nsf が更新されるまで検出されません。通常、このタスクは 夜間にスケジュール実行されています。

    カタログの表示

    Catalog を表示するには、Usage ウィンドウの最初のタブである Catalog を選択します。

    データベースの表示

    Catalog タブを選択した状態で、2 段目のタブから Databases タブを選択します。 これにより、カタログのデータベースビューが表示されます。

    このビューは 左右 2 つの領域に分かれています。左側には、すべてのデータベースと、それぞれの主要プロパティを一覧表示するテーブルがあります。データベースを選択すると、右側の詳細ペインにそのデータベースのすべてのACLを含むプロパティが表示されます。

    このテーブルは、Usage の他のすべてのテーブルと同様に、並べ替えおよび検索が可能です。列ヘッダーをクリックすると、その列を基準にテーブルが並べ替えられます。もう一度クリックすると逆順で並べ替えられ、さらにもう一度クリックすると並べ替えは解除されます。 テーブルをクリックした状態で Ctrl+F を押すと検索パネルが表示され、表示するデータベースを一時的に絞り込むことができます。なお、この検索機能は一覧の中から目的のデータベースを素早く見つけるためのものです。表示されるデータベース自体を制限したい場合は、以下の フィルター セクションを参照してください。

    一覧ペインと詳細ペインの相対的なサイズは変更可能で、その設定は保存されます。テーブルをより広く表示したい場合は、詳細ペインを完全に閉じることも可能です。再度表示するには、テーブル内の任意のデータベースをダブルクリックしてください。

    Teamstudio Usage は、すべてのカタログスキャン結果の完全な履歴を保持しているわけではありません。ただし、一度カタログに追加されたデータベースが削除されることはありません。これは、そのデータベースに関連する Usage データが存在する可能性があるためです。データベースが直近のカタログスキャンに存在していたかどうかは、データベースパスの横に表示されるサーバーアイコンを見ることで簡単に確認できます。このアイコンが表示されていない場合、そのデータベースは最新のサーバースキャンでは検出されていません。また、詳細ペインでは、直近のサーバースキャンにそのデータベースが存在していたかどうかを示すチェックボックスや、最後にスキャンで検出された日時も確認できます。

    データベース一覧には、catalog.nsf で一度も検出されたことのないデータベースが含まれる場合もありますが、これはまれなケースです。これは、Usage が Usage データのインポート時に未知のデータベースを検出した場合に発生することがあります。Usage データを関連付けるためにはカタログエントリが必要となるため、データベースのサーバー名とパスのみを含む'stub'エントリが作成されます。

    ユーザーの表示

    Catalog タブを選択した状態で、2 段目のタブから Users タブを選択します。これにより、カタログのユーザービューが表示されます。

    このビューの操作方法は、前述の Databases ビューとほぼ同様です。ただし、ユーザーについては、直近のスキャンで検出されたかどうかを示すサーバーアイコンは表示されません。これは、ユーザー名の存在を最終的に判断する単一のサーバーが存在しないためです。同じユーザーが異なるサーバー上の複数のアドレス帳に定義されている場合があり、あるサーバーには存在していても、別のサーバーには存在しないことがあります。

    ユーザー名とサーバー名

    Teamstudio Usage は、すべてのユーザー名およびサーバー名を内部的に完全な Canonical 形式（例：CN=Demo Adviser/O=Teamstudio）で保存します。UI では 共通名のみが表示されます（前述の例では Demo Adviser）。名前の上にマウスカーソルを置くと、省略形式のフルネーム（Demo Adviser/Teamstudio）を確認できます。

    フィルター

    フィルタリングを使用すると、さまざまな条件に基づいてデータベースやユーザーを UI 上から非表示にすることができます。Teamstudio Usage は、フィルタリングされたデータベースやユーザーについても Catalog データおよび Usage データの収集を継続するため、データを失うことなくいつでもフィルターの追加や削除を行うことができます。データベースまたはユーザーをフィルタリングするには、Catalog の該当するタブに切り替え、非表示にしたいエントリを右クリックして、ポップアップメニューから Ignore を選択します。

    これにより、選択した要素に応じたオプションの一覧が表示されます。ユーザーの場合は、その特定のユーザーを非表示にするオプションのみが表示されます。データベースの場合は、データベースのパス、テンプレート名、またはレプリカ ID に基づいて非表示にすることができます。フィルタリングの詳細については、フィルタリング￼ ページを参照してください。
